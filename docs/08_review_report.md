## 2. 수치/데이터 관련 핵심 문제점

### 2.1 근거 없는 수치 목표 (가장 심각)

| 문서 | 문제 수치 | 문제점 |
|---|---|---|
| 04 (보류) / 06 | AI 톡 일간 응답률 70% | 유사 서비스 벤치마크 근거 없음 |
| 04 (보류) / 06 | 톡→전화 전환율 65% | 완전 신규 기능이라 비교 대상 없음 |
| 04 (보류) / 06 | 표현 코칭 수용률 50% | 근거 없는 낙관적 수치 |
| 06 | 수신 거절률 40%→20% | 절반 감소의 정량적 근거 불충분 |
| 06 | AI 단독 유저 월 성장률 30~40% | 현재 10~20%에서 2~3배 성장의 근거 부재 |

### 2.2 시장 규모 추정 근거 부족

| 항목 | 현재 기술 | 문제점 |
|---|---|---|
| SOM 3,000~5,000억원 | "SAM의 20~25% 추정" | 20~25% 비율의 산출 근거 없음 |
| SAM 1.5~2조원 | "성인 영어교육의 35~40% 온라인" | 비율의 출처 없음 |
| TAM 4~5조원 | "교육시장의 8~10% 추정" | 비율 추정 근거 없음 |

### 2.3 경쟁사 데이터 검증 필요

| 데이터 | 현재 기술 | 문제점 |
|---|---|---|
| 헤이링 시장 영향력 | "초기 단계"로만 기술 | MAU/다운로드/매출 등 정량 데이터 전무 |
| 스픽 한국 월 매출 ~$300만 | Sensor Tower 인용 | 최신 데이터인지 확인 필요 (2024 vs 2026) |
| Duolingo 2025 매출 $10.4억 | 기술됨 | 실제는 2024 실적일 가능성 — 연도 표기 재확인 |
| AI 콜센터 만족도 20% | 아시아경제 2024 인용 | 원문 확인 시 수치가 다를 수 있음 |

### 2.4 내부 데이터 vs 외부 데이터 구분 미흡

| 데이터 | 분류 불명확 |
|---|---|
| 롤플레이 결제 전환율 2.4배 | 공모전 제공 데이터인지, 외부 리서치인지 혼재 |
| AI 발신 77.7% / 사용자 발신 22.3% | 출처 표기 불일치 |
| 1:1 유저 AI 병행률 50% | 공모전 가이드라인 vs 리서치 출처 구분 필요 |

## 3. 문서 간 일관성 문제

### 3.1 "AI 전화영어 경쟁사 전무" 표현 모순

- **03_problem_definition.md**: "경쟁사 중 AI 전화영어를 제공하는 곳은 없다"
- **01_market_analysis.md**: 헤이링을 "AI 전화영어 직접 경쟁자"로 분석
- **수정 방향**: "대형 경쟁사 중 AI 전화영어를 핵심 기능으로 제공하는 곳은 링글과 헤이링뿐이며, 프리미엄 비즈니스 영어 특화는 링글이 유일"로 수정

### 3.2 Daily Log vs AI 톡 이중 구조

- **03_problem_definition.md**: Daily Log를 P0-Core로 기술
- **04_feature_spec.md**: AI 톡이 Daily Log를 대체한다고 기술
- **05_wireframe_spec.md**: WF-07(Daily Log)과 WF-09(AI 톡) 둘 다 존재
- **수정 방향**: Daily Log를 "AI 톡 미사용 시 폴백 입력 옵션"으로 재정의. 메인 컨텍스트 수집은 AI 톡

### 3.3 Privacy 우선순위 불일치

- **04_feature_spec.md** (보류): P2로 분류
- **06_ppt_outline.md**: Privacy 관련 언급 없음
- **수정 방향**: 06에서 슬라이드 4-1b 또는 별도 슬라이드에 개인정보 보호 설계 1줄 언급 추가

## 4. 공모전 심사 관점 리스크

### 4.1 "외부 리서치/유저 스터디" 부족 (가장 큰 리스크)

CLAUDE.md에서 공모전이 기대하는 것:
> "링글 관점이 아닌 외부 리서치/유저 스터디/경쟁사 벤치마킹"

**현재 상태**: 앱스토어 리뷰 원문 인용 0건, 유저 인터뷰/설문 데이터 0건
**수정 방향**: 앱스토어 실제 리뷰 인용, 커뮤니티(에브리타임/블라인드) 의견 추가

### 4.2 실행 가능성 의문

- AI 톡 + 표현 코칭 + 맥락 추출 + 캐릭터 생성 + 스마트 거절 + 온보딩 개선을 6개월에 구현
- Phase 1 MVP 범위를 더 좁게 재정의 필요

### 4.3 AI 톡 차별화 근거 부족

- "AI와 채팅" 자체는 ChatGPT, 톡이즈 등 이미 존재
- 링글 AI 톡만의 차별점(아이비리그 데이터 기반 표현 코칭)을 더 명확히 서술 필요

## 5. 문서별 수정 지시사항

### 01_market_analysis.md (market-analyst)

1. **시장 규모 추정에 산출 근거 추가**: TAM/SAM/SOM 각 비율에 출처 또는 산출 로직 명시
2. **헤이링 정량 데이터 보완**: 앱스토어 순위, 리뷰 수, 다운로드 추정 등 확인 가능한 데이터 추가
3. **스픽/Duolingo 데이터 연도 명확화**: 각 수치가 몇 년 기준인지 재확인 및 표기
4. **섹션 0과 섹션 3~4 중복 정리**: 핵심 4개사 심층분석(섹션 0)을 메인으로, 이후 섹션은 중복 제거
5. **데이터 출처 신뢰도 등급 추가**: 각 데이터에 [공식 발표] [추정] [미확인] 태그

### 02_ringle_service_analysis.md (service-analyst)

1. **앱스토어 실제 리뷰 원문 인용 추가**: 링글/스픽/헤이링 앱스토어에서 실제 리뷰 3~5건 인용 (별점, 날짜 포함)
2. **수신 거절 가설에 "검증 방법" 컬럼 추가**: 각 가설별 검증 가능한 방법론 제시
3. **데이터 출처 구분 명확화**: [공모전 제공 데이터] vs [공개 자료 분석] vs [앱 실사용 확인] 태그 추가
4. **커뮤니티 기반 Pain Point 보강**: 블라인드/에브리타임 등 직장인/대학생 커뮤니티 의견 추가

### 03_problem_definition.md (Lead)

1. **"경쟁사 AI 전화영어 전무" 표현 수정**: 헤이링 존재 반영
2. **Problem #3 재포지셔닝**: 독립 문제가 아닌 #1, #2의 구조적 공통 원인으로 재정의
3. **3-Why 분석 Why 3 수정**: "솔루션 부재" → "사용자 맥락을 학습 재료로 전환하는 구조 부재"
4. **KPI 목표치에 보수적/낙관적 시나리오 병기**
5. **Daily Log를 AI 톡의 폴백 옵션으로 재정의**

### 05_wireframe_spec.md (ux-designer)

1. **WF-07(Daily Log)을 "AI 톡 미사용 시 폴백"으로 재정의**: 메인 플로우에서 제외
2. **WF-06 온보딩에서 "2.4배" 직접 노출 제거**: 사용자 지향적 메시지로 대체
3. **화면 번호 체계는 유지하되, 플로우 다이어그램에서 메인/서브 경로 명확히 구분**

### 06_ppt_outline.md (doc-writer)

1. **슬라이드 5-1 KPI 표에 근거 컬럼 보강**: 각 목표치의 산출 근거 또는 벤치마크 명시
2. **슬라이드 2-3 포지셔닝 맵에 헤이링 추가**
3. **슬라이드 4-1과 4-1b 통합 검토**: 정보량 압축
4. **슬라이드 4-6 와이어프레임을 2장 분리 검토**
5. **Privacy 관련 1줄 언급 추가** (슬라이드 4-1b 또는 5-4)
6. **발표 키포인트를 1문장으로 압축**
7. **KPI 목표치에 보수적/낙관적 구간 표기**

### 07_concept_board.md (doc-writer)

1. **좌측 영역 정보량 축소**: LCP 플로우 다이어그램을 우측 비주얼에 통합
2. **"LCP + RING LOOP" 대신 "RING LOOP" 또는 "AI 톡 + RING LOOP"로 단순화**
3. **핵심 수치를 3개에서 2개로 축소** (수신 거절률 + 결제 전환율)
4. **비주얼 제작 방법 명시** (AI 이미지 생성 도구 활용 시 AI 활용 명시 포함)
