## 🚨 최우선 체크리스트 (매 작업 전 확인)

- [ ] **하나의 서비스에만 집중**하고 있는가? (AI 튜터 OR AI 전화영어, 둘 다 X)
- [ ] **문제 정의가 한 문장**으로 명확한가?
- [ ] 문제 → 방향 → 기능 → 여정 → 성과의 **일관된 논리 흐름**이 있는가?
- [ ] 모든 기대 효과에 **6개월 내 측정 가능한 KPI**가 있는가?
- [ ] 제목·요약·컨셉보드를 **별도 산출물**로 정성스럽게 준비하고 있는가?
- [ ] **링글 기존 서비스 위에서** 설계하고 있는가? (기존 기능을 다시 만들고 있지 않은가?)
- [ ] AI 톡이 **대화 속에서 자연스럽게** 사용자 데이터를 수집하고 있는가? (설문/폼 X)
- [ ] 사용자가 AI 톡을 **쓰고 싶고 써야 할 명분**이 명확한가?

## 1. 평가 구조 이해

| 평가 축 | 배점 | 핵심 |
|---------|------|------|
| **내부 심사위원** | **90%** | 문제 정의 정확성, 논리 전개, 해결 일관성, 구현 가능성 |
| **확산 점수** | **10%** | 제목·요약·컨셉보드 → 블로그 공개 후 반응 (전달력 평가) |

> 심사위원은 **기획안 자체의 설계 품질**을 정교하게 평가한다.
> 확산 점수는 단순 좋아요 경쟁이 아니라 **"기획안의 전달력"** 평가이다.

---

## 2. 서비스 기획 부문 — 5단계 필수 구조

### ⭐ 가장 중요한 첫 번째 결정: "무엇을 다룰 것인가"

> **AI 튜터 OR AI 전화영어, 반드시 하나만 선택한다.**
> - 두 서비스는 목적과 사용 방식이 다름
> - 동시 진행 → 문제 정의 흐려짐, 기능 설계 얕아짐
> - 과거 수상작 분석: **"하나의 서비스, 하나의 문제, 하나의 흐름"**에 집중

### 📐 기획안 5단계 흐름

| 단계 | 내용 | 핵심 질문 |
|------|------|-----------|
| **1. 문제 정의** | 해결할 문제를 한 문장으로 규정 | 서비스의 어떤 지점이 유저 경험을 방해하는가? |
| **2. 개선 방향** | 큰 틀의 방향성 설정 | "발화량 증가", "온보딩 강화" 등 명확한 키워드 |
| **3. 기능 설계** | 화면 구조, 시스템 동작, 대화 흐름 구체화 | 사용자가 실제로 어떤 경험을 하게 되는가? |
| **4. 사용자 여정** | 문제 → 해결 → 개선된 경험 흐름 | 일관된 스토리라인이 있는가? |
| **5. 기대 효과** | 정량 지표 중심 성과 제시 | 6개월 내 측정 가능한 값인가? |

---

## 3. 심사위원이 보는 기획안의 조건

심사위원은 다음을 **순서대로** 확인한다:

1. ✅ 앞단에 **문제 정의**가 있는가 (단순 의견 X, 논리적 정리)
2. ✅ 문제 해결을 위한 **방향성**이 제시되었는가
3. ✅ **기능 설계·UI 구상·사용자 흐름**이 구체적으로 전개되었는가
4. ✅ 실제 **개선 효과**와 **6개월 내 측정 가능한 KPI**가 정리되었는가

> 💡 **"구현 가능성과 효과 예측"은 평가의 핵심 항목이다.**

---

## 4. 확산 점수 10점 — 3가지 필수 체크

### 제목
- ❌ 화려한 문구
- ✅ **"무엇을 해결하는 기획인가?"가 직관적으로 읽히는 구조**

### 요약 (3~4줄)
- 구성: **문제 – 핵심 해결 방식 – 기대 효과**
- 빠르게 이해되는 요약이 블로그에서 높은 반응 유도

### 컨셉보드 (A4 1매)
- ❌ 포스터 스타일의 화려함
- ✅ **문제·핵심 솔루션·기대 효과가 일목요연하게 정리된 요약형 카드**
- 핵심: **몇 초 안에 "이 기획이 무엇을 말하는지" 이해** 가능해야 함

---

## 5. 절대 하지 말아야 할 실수 🚫

| 실수 | 왜 감점되는가 |
|------|--------------|
| 두 서비스를 동시에 다루기 | 문제 정의가 흐려지고 기능 설계가 얕아짐 |
| AI 스피킹을 부가 기능으로만 제시 | 단독 상품 성장 전략이 핵심인데 벗어남 |
| 정성적 표현만 사용, 정량 지표 부재 | "구현 가능성과 효과 예측" 평가 불가 |
| 제목·요약·컨셉보드를 부수적 취급 | 확산 점수 10%에 직접 영향 |
| 6개월 초과 장기 계획 제시 | 실행 가능성 의문 → 감점 |
| 기존 화상영어 고객 크로스셀링 제안 | **공모전 제약사항 위반** |
| 글로벌 대규모 캠페인 제안 | 비현실적 → 실행 가능성 낮음 |

## 6. 우리 프로젝트 방향성 점검

### 현재 선택: AI 전화영어에 집중 (AI 톡은 솔루션 수단)

> ✅ **"하나의 서비스, 하나의 문제, 하나의 흐름" 원칙 충족**
> - 집중 대상: AI 전화영어 (수신 거절률 40%)
> - 솔루션: AI 톡이 페르소나 기반으로 전화 전 관계·맥락 형성 (RING LOOP)
> - AI 톡은 독립 서비스 개선이 아닌, 거절률 해결을 위한 **수단**

### 점검 항목

| 항목 | 현재 상태 | 가이드 기준 | 상태 |
|------|-----------|------------|------|
| 서비스 집중도 | AI 전화영어 집중, AI 톡은 수단 | 하나만 선택 권장 | ✅ 충족 |
| 문제 정의 | "거절률 40%, 준비 안 됐기 때문" | 한 문장으로 명확해야 | ✅ 완료 |
| 5단계 논리 구조 | 문제→원인→RING LOOP→여정→KPI | 문제→방향→기능→여정→성과 | ✅ 완료 |
| KPI 정량 지표 | 거절률·발신·전환율 수치 있음 | 6개월 내 측정 가능한 값 | ✅ 완료 |
| 컨셉보드 | docs/07 초안 | 요약형 카드, 몇 초 내 이해 | ⚠️ 확인 필요 |
| 제목·요약 | 미정 | 직관적 전달력 | 🔴 작성 필요 |
