## 2. 핵심 KPI 목표 — 타당성 근거

### 2.1 수신 거절률: 40% → 25% (보수적) / 20% (낙관적)

**현재 상태**: 약 40% (링글 공식 제공)

**목표 산출 논리**:

| 개선 메커니즘 | 예상 기여도 | 근거 |
|---|---|---|
| **AI 톡 예열 (Ready)** | 거절률 8~10%p 감소 | 톡으로 사전 예열 시 "갑작스러움(H2)" + "맥락 부재(H3)" 동시 해소. 사용자가 이미 주제를 알고 전화를 받으므로 심리적 준비 완료 |
| **스마트 거절 UX** | 거절률 5~7%p 감소 | "5분 뒤 재발신", "내가 먼저 전화", "오늘은 건너뛸게" 옵션으로 순수 거절을 재스케줄링으로 전환 |
| **합산 예상** | 13~17%p 감소 → 거절률 23~27% | 보수적 25% / 낙관적 20% |

**외부 벤치마크 근거**:

| 벤치마크 | 데이터 | 출처 | 시사점 |
|---|---|---|---|
| AI 콜센터 거부감 | **73.6%** | 아시아경제 (2024.08) | 맥락 없는 AI 전화에 대한 일반적 거부감. 현재 40%는 이미 "자발적 예약"이라 일반 대비 낮음. 사전 맥락 제공으로 추가 개선 여지 큼 |
| AI 콜센터 만족도 | **20%** | 아시아경제 (2024.08) | 맥락 없는 AI 발신의 한계. 맥락 제공 시 만족도 상승 → 거절률 하락 가능 |
| 헤이링 사용자 리뷰 | "전화 올 때 무슨 주제인지 몰라서 당황" | App Store (2026) | 경쟁사도 동일 문제 인식. 사전 주제 제공만으로도 개선 효과 입증 가능성 |
| 사전 알림 효과 (의료분야 유사사례) | 예약 리마인더로 no-show 25~40% 감소 | 다수 의료 연구 메타분석 | 사전 알림/준비가 불이행률을 크게 낮추는 것은 다양한 분야에서 검증된 패턴 |

**보수성 검증**:
- 현재 거절률 40% 중 **환경적 제약(H4: 회의중/이동중)**에 의한 불가피한 거절은 약 10~15%로 추정
- 따라서 심리적/UX적 원인에 의한 거절은 25~30%p
- 이 중 절반만 해소해도 12~15%p 감소 → **25% 목표는 달성 가능한 보수적 수치**

### 2.2 결제 전환율: 롤플레이 경험자 기준 2.4배

**현재 상태**: 롤플레잉 사용자 결제 전환율이 미사용자 대비 2.4배 (링글 내부 데이터)

**이 수치의 의미**:
- 이 데이터는 **기존 링글 사용자 코호트 분석** 결과로, 공모전에서 공식 제공한 1차 데이터
- 우리 기획의 "달성 목표"가 아니라 **"이미 검증된 사실"을 활용하는 전략**

**기획에서의 활용 논리**:

```
[전제] 롤플레이 경험 → 결제 전환율 2.4배 (검증된 사실)
      ↓
[전략] 모든 사용자가 롤플레이를 경험하도록 설계
      - 온보딩: 레벨테스트 직후 3분 롤플레잉 자동 연결 (MVP-05)
      - AI 톡: 대화 → 롤플레이 시나리오 자동 생성 → 전화 연결 (MVP-01~03)
      ↓
[목표] 롤플레잉 온보딩 경험률 70~80%로 끌어올리면
       → 전체 결제 전환율도 자연스럽게 상승
```

**외부 벤치마크**:

| 벤치마크 | 데이터 | 출처 | 시사점 |
|---|---|---|---|
| Duolingo 온보딩 완료율 | **~70%** | Duolingo IR, 업계 분석 | 잘 설계된 온보딩의 완료율 기준. 우리의 70% 목표와 일치 |
| SaaS "AHA Moment" 도달 시 전환율 | 일반적으로 2~5배 | 다수 SaaS 연구 (Mixpanel, Amplitude) | 핵심 기능 체험 후 전환율이 2~5배 높아지는 것은 업계 표준 패턴. 링글의 2.4배는 이 범위 안에 위치 |
| 스픽 라이브 롤플레이 도입 후 | 매출 전년비 2배+ 성장 | Sensor Tower (2024 상반기) | 롤플레이 기능이 매출에 미치는 직접적 영향 |

**보수성 검증**:
- 2.4배는 링글이 이미 관측한 실제 데이터 (추정치가 아님)
- 우리 기획은 이 수치를 "만들어내는" 것이 아니라, **더 많은 사용자가 이 경험에 도달하도록 유도하는 구조**를 설계
- 보수적 목표(2.0배)는 온보딩 마찰에 의한 경험 품질 저하를 감안한 하향 조정치

### 2.3 사용자 발신 비율: 22.3% → 30% (보수적) / 35% (낙관적)

**현재 상태**: 22.3% (링글 공식 제공. AI 발신 77.7%의 보수)

**목표 산출 논리**:

| 개선 메커니즘 | 예상 기여 | 근거 |
|---|---|---|
| **스마트 거절 UX "내가 먼저 전화할게요" 옵션** | 발신 비율 +4~6%p | 거절 시 "사용자 발신" 대안 제공. 거절 40% 중 15~20%가 이 옵션 선택 시 약 6%p 증가 |
| **AI 톡에서 자발적 전화 요청** | 발신 비율 +2~4%p | 톡 대화 중 "전화해줘"라고 사용자가 먼저 요청하는 패턴 유도 |
| **합산** | +6~10%p → 28~32% | 보수적 30% / 낙관적 35% |

**산출 세부**:
```
현재 AI 발신 건수 = 전체의 77.7%
현재 거절률 = 40% → 거절 건수 = 전체 × 77.7% × 40% = 전체의 31.1%

스마트 거절에서 "내가 먼저 전화" 선택률 추정: 15~20%
→ 전환 건수 = 전체의 31.1% × 17.5%(중간값) = 전체의 5.4%

기존 사용자 발신 22.3% + 전환분 5.4% = 27.7%
+ AI 톡 자발적 요청 2~4% = 약 30~32%

→ 보수적 목표 30%는 산술적으로 타당
```

### 2.4 온보딩 롤플레잉 경험률: 추정 30~40% → 70% (보수적) / 80% (낙관적)

**현재 상태**: 정확한 수치 미공개. "이탈 구간 = 가입 직후"이고 롤플레잉이 탐색적 접근에 의존 → 30~40%로 추정

**목표 산출 논리**:

| 개선 | 예상 효과 | 근거 |
|---|---|---|
| **레벨테스트 직후 자동 롤플레잉 연결** | 경험률 +25~35%p | 선택이 아닌 기본 경로로 배치. "건너뛰기"는 우측 상단 작게 제공 |
| **3분 체험** 시간 제한 | 이탈 방지 | "3분이면 끝"이라는 심리적 부담 최소화 |

**외부 벤치마크**:

| 벤치마크 | 데이터 | 출처 | 시사점 |
|---|---|---|---|
| Duolingo 온보딩 첫 레슨 완료율 | **~70%** | Duolingo IR 분석 | 기본 경로로 설계된 온보딩의 달성 가능한 상한 |
| 모바일 앱 온보딩 평균 완료율 | 40~60% | 업계 표준 (Appsflyer) | 일반적 앱 기준. 잘 설계된 경우 70%+ 가능 |
| 링글 레벨테스트 완료율 | 높을 것으로 추정 | 가입 플로우 필수 단계 | 레벨테스트 바로 뒤에 배치하면 이탈 전 포착 가능 |

**보수성 검증**:
- 70%는 Duolingo 수준의 최적화된 온보딩 기준
- 링글은 영어학습에 이미 동기부여된 사용자(유료 앱 다운로드)이므로 일반 앱 대비 높은 완료율 예상
- 80%(낙관적)는 최적 UX + 높은 동기부여 조건에서 달성 가능

## 5. 수치 간 교차 검증

### 5.1 거절률 40% → 25% 달성 시 파급 효과

```
현재 상태:
- 월 AI 전화 예약 100건 기준
- 정상 연결: 60건 (60%)
- 거절: 40건 (40%)

개선 후 (거절률 25%):
- 정상 연결: 75건 (75%) → +25% 학습 세션 증가
- 거절: 25건 (25%)

→ 월간 학습 세션 25% 증가
→ 습관 형성 확률 상승 → 리텐션 개선 → LTV 증가
```

### 5.2 롤플레이 경험률 70% × 2.4배 전환율의 매출 임팩트

```
현재 추정:
- 무료 체험 유저 중 롤플레이 경험률: ~35%
- 롤플레이 경험자 전환율: X × 2.4
- 미경험자 전환율: X

개선 후:
- 경험률: 70% (2배)
- 경험자 풀이 2배 → 높은 전환율(2.4×) 적용 대상 2배
- 전체 전환율 추정: X × (0.70 × 2.4 + 0.30 × 1.0) = X × 1.98
  (현재: X × (0.35 × 2.4 + 0.65 × 1.0) = X × 1.49)

→ 전체 결제 전환율 약 33% 상승 예상
```

### 5.3 사용자 발신 30% 달성의 의미

```
현재: AI 발신 77.7% / 사용자 발신 22.3%
목표: AI 발신 70% / 사용자 발신 30%

사용자 발신 = 능동적 학습 의지
→ 발신 비율 +7.7%p는 "AI 푸시 의존 → 자발적 학습 루틴"으로의 전환 지표
→ 리텐션 직결: 스스로 전화하는 사용자는 이탈률이 낮음
```
