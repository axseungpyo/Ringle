## 개요

### LCP (Learner Context Platform)
사용자의 일상/업무 컨텍스트를 **AI 톡 대화**를 통해 자연스럽게 수집하고,
AI가 학습 재료로 자동 변환하여 AI 튜터 · AI 전화영어 · 1:1 튜터 수업 세 채널에 동일하게 공급하는 **컨텍스트 레이어**.

AI 톡에서 수집된 컨텍스트는 LCP의 Context Transformer가 자동으로 Phone Call Brief, Roleplay Script 등 학습 재료로 변환하여 각 채널에 공급한다.

```
[Context Capture]            [Context Transformer]       [Delivery]
AI 톡 대화 (안부/근황)  →   Topic Card 자동 생성    →  AI 튜터 (롤플레잉/디스커션)
약점/습관 패턴 분석          Phone Call Brief 생성       AI 전화 (RING LOOP)
                             Roleplay Script 생성        1:1 튜터 (Tutor Brief)
                             Tutor Brief 생성
```

### RING LOOP (LCP의 AI 전화영어 구현 플로우)
LCP가 생성한 Phone Call Brief를 AI 전화영어에 적용하는 구체적 UX 루프.
**AI 톡이 Ready 단계의 핵심 인터페이스**로, 자연스러운 대화를 통해 전화를 예열한다.

```
[Ready]                   [In]                     [Grow]
AI 톡 대화 (안부/근황)   맥락형 AI 전화 수신   →  톡으로 피드백 전달
톡에서 전화 자연 전환  →  롤플레이 캐릭터 대화      톡 피드백 전달 (Phase 2 확장)
                          실시간 힌트 카드
```

## Phase 1: MVP 핵심 기능 (3개월)

---

### MVP-01. AI 톡 — 일상 대화 + 컨텍스트 수집

#### 기능 목적
AI가 **친구처럼 톡(채팅)으로 안부와 근황을 물어보면서** 사용자의 일상/업무 컨텍스트를 자연스럽게 수집한다.
폼 입력이 아닌 **대화형 인터페이스**로 입력 부담을 제거하고, 학습이 일상 대화 속에 녹아든다.

#### 해결하는 Pain Point
- **결정 피로**: 콘텐츠가 많을수록 주제 선택 부담 → 톡 대화에서 주제가 자연 도출
- **맥락 파편화**: AI 튜터/AI 전화/1:1 수업이 각각 다른 주제로 운영 → 톡에서 수집된 맥락이 세 채널 관통
- **외부 주제 거리감**: 미리 만들어진 시나리오가 "내 상황"과 맞지 않음 → 톡에서 나온 실제 이야기가 학습 재료
- **입력 부담**: 폼/로그 직접 작성 → 대화하듯 답하면 끝

#### Before / After 비교

| 구분 | Before (현재) | After (AI 톡 적용) |
|---|---|---|
| 컨텍스트 수집 | 없음 (매번 카탈로그에서 주제 선택) | AI 톡이 안부를 물으며 자연 수집 |
| 사용자 입력 방식 | 폼 기반 직접 입력 | 친구와 톡하듯 대화 (한글 OK) |
| 영어 학습 시작점 | 앱 열기 → 주제 탐색 → 결정 → 학습 | 톡 답장 → 즉시 학습 연결 |
| 전화 전 경험 | 예고 없이 전화벨 | 톡으로 안부 → "이따 전화할게" 자연 전환 |
| 학습 연속성 | 세션별 단절 | 톡 대화가 계속 이어지며 맥락 축적 |

#### 상세 동작 방식

```
[오후 6시] AI 톡 도착
  AI: "Hey! How was your day? 오늘 회의 어떻게 됐어?"

  사용자: "아 오늘 발표했는데 영어로 질문 받아서 당황했어"

  AI: "Oh no! What was the question about?"
  사용자: "프로젝트 일정 관련이었어"

  AI: "That's tough! Want to practice handling Q&A situations?
       이따 9시쯤 전화로 연습해볼까?"

  사용자: [좋아! / 시간 변경 / 오늘은 패스]
```

**자동 추출되는 컨텍스트:**
- 상황: 업무 발표 (클라이언트 대면)
- 언어 니즈: Q&A 대응 표현
- 감정: 좌절/아쉬움
- 긴급도: 높음 (반복 상황 가능성)
- 직무 키워드: 프로젝트 일정, 발표
- 약점 태그: #프레젠테이션QA #시간표현

#### 기대 KPI

| 지표 | 보수적 목표 | 낙관적 목표 | 측정 방법 |
|---|---|---|---|
| AI 톡 일간 응답률 | 활성 유저의 60% | 70% | 일간 톡 응답 유저 수 / 활성 유저 수 |
| 톡 → 전화 전환율 | 55% | 65% | 톡 대화 시작 → 전화 수신/발신 전환율 |
| 홈 화면 이탈률 감소 | 20% 감소 | 30% 감소 | A/B 테스트 (톡 적용 전후 비교) |

#### 기술 구현 가능성

- **채팅 인터페이스**: 앱 내 채팅 UI (표준 모바일 패턴) + 서버 메시지 스케줄링
- **자연어 대화**: GPT-4o/Claude API 기반 대화 생성 (현 상용 LLM으로 구현 가능)
- **컨텍스트 추출**: LLM에 대화 히스토리 입력 → 구조화된 태그/키워드 자동 추출

### MVP-02. AI 톡 → 전화 자연 전환 (Ready 단계)

#### 기능 목적
전화 수신 전, **AI 톡으로 안부와 근황을 나누면서 자연스럽게 전화로 이어지도록** 설계하여
**"갑자기 전화가 온다"는 심리적 부담**과 **"뭘 말해야 할지 모른다"는 맥락 부재**를 동시에 제거한다.
친구와 카톡/인스타DM 하다가 "전화할게~" 하는 자연스러운 흐름을 AI가 구현한다.

#### 해결하는 Pain Point
- **H2 갑작스러움에 의한 부담**: 예약했더라도 실제 전화벨이 울리면 회피 반응 → 톡으로 예열되어 있으므로 자연스러운 수신
- **H3 맥락 부재**: 어떤 주제로 대화할지 사전 정보 없이 전화가 옴 → 톡에서 이미 주제가 정해짐
- **U2 사전 맥락 미제공**: 전화 수신 전 워밍업 없음 → 톡 대화 자체가 워밍업

#### Before / After 비교

| 구분 | Before (현재) | After (AI 톡 전환) |
|---|---|---|
| 전화 전 경험 | 아무런 알림 없음 → 전화벨이 갑자기 울림 | 톡으로 안부 → 주제 자연 도출 → "전화할게" 전환 |
| 주제 인지 | 전화 연결 후에야 AI가 주제 언급 | 톡에서 이미 오늘 이야기를 나눔 → 주제 사전 인지 |
| 심리 상태 | "갑자기? 지금 바쁜데..." → 거절 | "아 오늘 그 얘기로 연습하는 거구나" → 수신 |
| 사용자 선택권 | 수신 또는 거절(무응답) 2가지 | 톡에서 "좋아!" / "시간 변경" / "오늘은 패스" 선택 |
| 컨텍스트 활용 | 없음 | 톡 대화가 곧 전화 시나리오 소스 |

#### 상세 동작 방식 (단계별)

**Step 1 — AI 톡 대화 시작 (전화 예약 시간 1~3시간 전)**
- 사용자가 설정한 AI 전화 예약 시간 기준 1~3시간 전, AI가 톡으로 안부 인사
- 시간대별 자연스러운 인사:
  - 오전 (07~09시 예약): 전날 저녁 또는 당일 아침 "Good morning! Any plans for today?"
  - 저녁 (21~23시 예약): 오후 퇴근 무렵 "Hey! How was your day?"
- **[LCP 연계]** 개인화 수준(Lv.1~4)에 따라 인사 내용이 다름

**Step 2 — 대화 중 컨텍스트 수집**
- 사용자가 오늘 있었던 일, 고민, 내일 일정 등을 자연스럽게 공유
- AI가 공감하면서 영어 관련 니즈를 자연스럽게 파악
- Phase 2에서 표현 코칭 추가 예정

**Step 3 — 전화 전환 제안**
- 대화 흐름에서 자연스럽게 전화 제안:
  ```
  AI: "That's tough! Want to practice handling Q&A situations?
       이따 9시쯤 전화로 연습해볼까?"

  [좋아!] → 전화 예약 확정 + 시나리오 자동 생성
  [시간 변경] → 시간 피커 등장
  [오늘은 패스] → "괜찮아! 내일 또 톡할게"
  ```
- 사용자가 직접 "전화해줘"라고 먼저 요청할 수도 있음 → 사용자 발신 비율 상승

**Step 4 — 전화 직전 최종 브리핑 (톡 내)**
- 전화 10분 전, 톡으로 간단한 브리핑 전달:
  ```
  AI: "10분 뒤에 전화할게! 오늘 연습 내용 미리 보기:
       Topic: Deadline Extension Request
       Key phrase: 'I'd like to discuss a timeline adjustment'
       Ready?"

  [준비됐어!] [5분만 더!]
  ```

**Step 5 — 전화 연결**
- 톡에서 이미 대화한 맥락 그대로 전화 시작
- AI 캐릭터가 톡에서 예고한 시나리오로 바로 진입

#### 기대 KPI 및 측정 지표

| KPI | 현재 | 보수적 목표 | 낙관적 목표 | 측정 방법 |
|---|---|---|---|---|
| AI 전화 수신 거절률 | 약 40% | 25% | 20% | 톡 대화 후 수신률 vs 톡 미사용 수신률 A/B 테스트 |
| 톡 → 전화 전환율 | - (신규 기능) | 55% | 65% | 톡 대화 시작 → 전화 수신/발신 전환율 |
| 사용자 발신 비율 | 22.3% | 30% | 35% | 톡에서 "전화해줘" 직접 요청 비율 추적 |
| 전화 주제 만족도 | - | 3.8/5.0 | 4.2/5.0 | 통화 후 "오늘 주제가 내 상황에 맞았나요?" 설문 |

#### 기술 구현 가능성

- **채팅 인터페이스**: 앱 내 채팅 UI (표준 모바일 패턴) + 서버 메시지 스케줄링
- **자연어 대화**: GPT-4o/Claude API 기반 대화 생성 (현 상용 LLM으로 구현 가능)
- **컨텍스트 추출**: LLM에 대화 히스토리 입력 → 구조화된 태그/키워드 자동 추출
- **전화 전환**: 톡 내 CTA 버튼 → 기존 AI 전화영어 발신 API 연결

### MVP-03. 맥락형 AI 전화 롤플레이 (축소)

#### 기능 목적
AI 톡에서 나눈 대화 맥락이 그대로 전화로 이어져,
**"AI와 대화하는 어색함"을 캐릭터 몰입으로 완화**하고
**전화 = 롤플레이 연습**이라는 명확한 맥락을 제공한다.

#### 해결하는 Pain Point
- **H3 맥락 부재**: 전화 연결 후에도 어색한 AI 대화 시작 ("Hello! How are you today?")
- **앱스토어 불만 "AI와 대화하는 느낌이 가장 강하다. 자연스럽지 않다"**
- 롤플레이 사용자 결제 전환율 2.4배 데이터 → 전화영어에 롤플레이 경험을 결합하여 전환율 제고

#### Before / After 비교

| 구분 | Before (현재) | After (개선) |
|---|---|---|
| 전화 시작 | "Hello! I'm your Ringle AI tutor. How are you today?" | "Hi! I'm Jennifer, your interviewer from [회사명]. Thanks for joining us today." |
| 대화 맥락 | 프리토킹 형식, 주제 없이 자유 대화 | 톡에서 나눈 이야기가 시나리오로 연결된 롤플레이 |
| AI 역할 | 중립적 AI 튜터 | 면접관 / 외국인 동료 / 해외 클라이언트 등 구체적 캐릭터 |
| 대화 흐름 | AI가 주도하는 일방적 Q&A | 시나리오 구조 기반 자연스러운 양방향 대화 |
| 힌트 제공 | 통화 중 힌트 없음 | 통화 화면 하단 힌트 카드 (선택적 탭) |

#### 상세 동작 방식 (단계별)

**[LCP 연계]** 롤플레이 시나리오는 AI 톡 대화에서 추출된 컨텍스트 + Context Transformer가 생성한 Roleplay Script를 사용한다.

**Step 1 — 캐릭터 설정 (전화 연결 직전)**
- 톡 대화에서 도출된 시나리오 데이터가 AI 전화 세션에 전달됨
- AI는 해당 시나리오의 캐릭터 페르소나로 세팅:
  - 면접관 → 이름/직함/회사명 부여 (예: "Jennifer Park, HR Director at Mango Corp")
  - 외국인 동료 → 이름/역할 부여 (예: "Tom, Product Manager from the US HQ")
  - 해외 클라이언트 → 이름/업종 부여 (예: "Mr. Smith, VP at BlueKey Partners")

**Step 2 — 시나리오 기반 대화 진행**
- 시나리오는 4~6개 대화 스테이지로 구성:
  1. Opening (AI 캐릭터 인사 + 상황 설명)
  2. Core Topic 1~3 (시나리오 핵심 대화)
  3. Follow-up (사용자 발화 내용에 반응하는 심화 질문)
  4. Closing (자연스러운 마무리)
- 각 스테이지에서 사용자 발화 내용을 실시간으로 반영하여 응답 생성

**Step 3 — 통화 중 힌트 카드 (선택적)**
- 통화 화면 하단에 접힌 형태로 힌트 카드 탭 제공
- 탭하면 현재 스테이지에 맞는 유용한 표현 1~2개 표시
- 힌트를 본 후 발화 시 "힌트 사용" 표시 → 리포트에 반영

**Step 4 — 자막 옵션 유지**
- 기존 "영어 자막만 / 번역 포함 / 없음" 옵션 유지

#### MVP 롤플레이 캐릭터 & 시나리오 (3종)

| AI 캐릭터 | 시나리오 | 타겟 |
|---|---|---|
| 면접관 (HR Director) | 면접 자기소개 + 역량 질문 | 취업/커리어 |
| 외국인 동료 (Product Manager) | 주간 업무 보고 + 비즈니스 미팅 | 비즈니스 미팅 |
| 해외 클라이언트 (VP) | 프로젝트 제안 + 협상 | 프레젠테이션/협상 |

> 커스텀 캐릭터 및 추가 시나리오는 Phase 2에서 확장 예정

#### 기대 KPI 및 측정 지표

| KPI | 현재 | 보수적 목표 | 낙관적 목표 | 측정 방법 |
|---|---|---|---|---|
| 통화 당 평균 발화 단어 수 | 기준 미확인 | 120단어 이상 비율 60% | 150단어 이상 비율 70% | 통화 로그 분석 |
| 통화 완료율 (중도 포기 없이 완료) | 기준 미확인 | 70% | 80% | 세션 완료 이벤트 추적 |
| 힌트 카드 사용률 | - (신규) | 30% | 40% 이하 (너무 높으면 의존 위험) | 힌트 탭 이벤트 추적 |
| 롤플레이 전화 후 다음 전화 예약률 | 기준 미확인 | 55% | 65% | Grow 리포트 → 다음 예약 전환율 |

#### 기술 구현 가능성

- **캐릭터 페르소나 설정**: GPT-4o/Claude API의 System Prompt로 캐릭터 주입 → 현 상용 LLM으로 구현 가능
- **시나리오 기반 대화 흐름**: 시나리오 구조를 JSON으로 정의 → LLM에 컨텍스트로 주입하여 진행 방향 유도
- **실시간 음성 인식**: 기존 AI 전화영어 음성인식 인프라 재활용 (신규 R&D 불필요)
- **힌트 카드**: 시나리오별 핵심 표현 DB에서 현재 스테이지 매핑 → 정적 텍스트 제공 (LLM 호출 최소화)

### MVP-04. 스마트 거절 UX

#### 기능 목적
AI 전화 거절이 **"학습 중단"이 아닌 "재스케줄링"이 되도록** UX를 재설계하여
거절 → 이탈 → 방치 사이클을 차단하고 학습 연속성을 유지한다.

#### 해결하는 Pain Point
- **U3 예약 변경 번거로움**: 상황 변화 시 간편한 시간 변경/5분 지연 기능 부재 → 차라리 거절 선택
- **U4 거절 후 재시도 부재**: 거절 시 재발신 로직이나 "5분 뒤 다시 전화할까요?" 옵션 없음
- **H4 환경적 제약**: 전화 예약 시점과 실제 수신 시점의 상황 불일치

#### Before / After 비교

| 구분 | Before (현재) | After (개선) |
|---|---|---|
| 거절 버튼 | 통화 거절(빨간 버튼) 하나 | 거절 탭 시 스마트 옵션 바텀 시트 등장 |
| 거절 후 처리 | 연결 종료 → 학습 없음 → 자책감 | 옵션에 따라 재스케줄링 / 사용자 발신 / 건너뛰기 |
| 자책감 UX | 거절 후 앱이 조용해짐 → 방치 유발 | 톡으로 "괜찮아요! 내일 다시 톡할게요" 메시지 |
| 건너뛰기 후 | 오늘 학습 기록 없음, 스트릭 끊김 | 미니 대화 1~2턴으로 "학습 완료" 처리 옵션 제공 |

#### 상세 동작 방식 (단계별)

**Step 1 — 수신 화면에서 거절 탭 감지**
- 사용자가 수신 화면의 "거절" 버튼 탭 시 즉시 전화 연결 없이 바텀 시트 등장

**Step 2 — 스마트 거절 옵션 바텀 시트**
```
┌─────────────────────────────────────┐
│  지금은 좀 어렵네요. 어떻게 할까요?   │
│                                     │
│  [5분 뒤 다시 전화해줘]              │
│  [오늘은 건너뛸게요]                │
│  [내가 먼저 전화할게요]              │
│                                     │
│  [취소]                              │
└─────────────────────────────────────┘
```

**Step 3 — 옵션별 분기 처리**

| 옵션 | 즉시 처리 | 후속 UX |
|---|---|---|
| 5분 뒤 다시 전화해줘 | 서버 발신 예약 5분 지연 | 톡으로 "5분 뒤 다시 전화할게요" 메시지. 5분 후 재발신 |
| 오늘은 건너뛸게요 | 오늘 AI 전화 일정 취소 | 톡으로 "오늘은 쉬어가도 괜찮아요! 그래도 1분 톡 롤플레이 해볼까요?" CTA |
| 내가 먼저 전화할게요 | 앱 내 "바로 전화하기" 버튼으로 이동 | 사용자가 원할 때 직접 발신 → 사용자 발신 비율 22.3% → 35% 목표 |

**Step 4 — "오늘은 건너뛸게요" 선택 후 톡 미니 롤플레이 제안**
- 푸시 거절이 연속 3회 이상 발생하는 유저에게 특별 처리
- 톡에서: "오늘 3번 연속으로 전화를 못 받으셨네요. 톡에서 1분 영어 대화로 스트릭 지켜드릴까요?"
- 미니 톡 롤플레이 완료 시 스트릭 유지 처리 → 자책감 없는 재참여 유도

**Step 5 — 연속 거절 패턴 감지 및 예약 시간 최적화 제안**
- 특정 시간대에 거절이 반복되면 7일 후 톡으로:
  "21시에 전화를 자주 못 받으셨어요. 22시나 07시로 변경해볼까요?"

#### 기대 KPI 및 측정 지표

| KPI | 현재 | 보수적 목표 | 낙관적 목표 | 측정 방법 |
|---|---|---|---|---|
| AI 전화 수신 거절률 | 약 40% | 25% | 20% | 월별 거절률 추적 |
| 사용자 발신 비율 | 22.3% | 30% | 35% | 발신 방향 이벤트 로그 |
| "5분 뒤" 선택 후 실제 수신 전환율 | - (신규) | 65% | 75% | 지연 후 수신 이벤트 추적 |
| 톡 미니 롤플레이 수락률 | - (신규) | 30% | 40% | "건너뛰기" 후 톡 미니 롤플레이 CTA 전환율 |

#### 기술 구현 가능성

- **바텀 시트 UX**: 표준 모바일 UI 패턴으로 즉시 구현 가능
- **5분 지연 발신**: 서버 발신 스케줄러 파라미터 조정으로 구현 가능 (기존 예약 로직 재활용)
- **사용자 발신 연결**: 기존 "바로 전화하기" 기능으로 딥링크 처리
- **톡 미니 롤플레이**: AI 톡 채팅 인터페이스 내에서 1~2턴 영어 대화 진행
- **연속 거절 패턴 감지**: 거절 이벤트 로그 기반 간단한 룰 기반 로직

### MVP-05. 롤플레잉 온보딩

#### 기능 목적
앱 설치 후 레벨테스트 직후, 롤플레잉을 **"선택"이 아닌 "기본 경로"**로 배치하여
**AHA Moment**를 첫 3분 안에 발생시키고 결제 전환율을 극대화한다.

#### 해결하는 Pain Point
- 무료 체험 7일 중 롤플레잉 미경험 → 전환 실패
- 기존 온보딩: 기능 소개 위주 → 가장 강력한 기능(롤플레이)에 도달 전 이탈
- 롤플레이 경험자 결제 전환율 2.4배 높음 → 첫 경험으로 반드시 유도해야 함

#### Before / After 비교

| 구분 | Before (현재) | After (개선) |
|---|---|---|
| 온보딩 플로우 | 앱 설치 → 레벨테스트 → 홈 화면 (기능 탐색) | 앱 설치 → 레벨테스트 → 즉시 롤플레잉 3분 체험 |
| 롤플레이 도달 | 스스로 탐색해서 찾아야 함 | 레벨테스트 완료 직후 자동 연결 |
| 첫 5분 경험 | 기능 리스트 구경 → "뭐부터 해야 하지?" | 면접관 AI와 실제 대화 → "오, 이거 진짜 실전이네!" |
| AHA Moment | 발생 불확실 (타이밍 없음) | 첫 롤플레잉 완료 직후 결제 CTA 노출 (타이밍 명확) |

#### 상세 동작 방식 (단계별)

**Step 1 — 레벨테스트 완료 후 즉시 전환**
- 레벨테스트 결과 화면에서 3초 후 자동으로 "첫 롤플레잉 시작" 화면으로 전환
- (건너뛰기 버튼 우측 상단에 작게 제공 — 강제하지 않으나 기본 흐름)

**Step 2 — 시나리오 선택 (레벨 기반 자동 추천)**
- 레벨 결과에 맞는 시나리오 1개 자동 추천 (예: 초급 → "자기소개 / 면접관", 중급 → "비즈니스 미팅 보고")
- "다른 시나리오 보기" 링크로 전체 카탈로그 접근 가능

**Step 3 — 3분 롤플레잉 체험**
- 일반 롤플레잉과 동일하지만 "첫 체험 모드"로 진행:
  - 첫 스테이지에서 AI가 체험 가이드 음성 안내
  - 힌트 카드 기본 ON (체험 유저는 힌트 의존도 높아도 OK)
  - 3분 타이머 표시 (짧은 시간 → 심리적 부담 최소화)

**Step 4 — AHA Moment 설계: 롤플레잉 완료 직후**
- 완료 축하 화면 + 발화 통계 (발화량, 대화 턴, 힌트 사용)
- 7일 무료 체험 챌린지 설정 (롤플레잉 3회 완료 목표)
- 결제 CTA 노출

**Step 5 — AI 톡 첫 대화 유도**
- AHA Moment 화면 후 AI 톡 첫 대화 시작:
  "방금 롤플레잉 어땠어요? AI 톡에서 매일 영어 대화 연습해봐요!"
  → AI 톡 습관 형성의 출발점

#### 기대 KPI 및 측정 지표

| KPI | 현재 | 보수적 목표 | 낙관적 목표 | 측정 방법 |
|---|---|---|---|---|
| 온보딩 중 롤플레잉 경험률 | 기준 미확인 (낮을 것으로 추정) | 70% | 80% | 레벨테스트 완료 → 롤플레잉 시작 전환율 |
| 첫 롤플레잉 완료율 | - | 60% | 70% | 롤플레잉 시작 → 완료 전환율 |
| 결제 전환율 (무료 체험 중) | 기준 미확인 | 롤플레잉 3회 완료 유저의 결제 전환율 2.0배 | 2.4배 달성 | 코호트별 분석 |
| 온보딩 후 AI 톡 첫 대화 시작률 | - (신규) | 40% | 50% | AHA Moment → AI 톡 진입 전환율 |

#### 기술 구현 가능성

- **온보딩 플로우 재설계**: 기존 레벨테스트 완료 후 라우팅 로직 변경 → 중간 복잡도
- **체험 모드 롤플레잉**: 기존 롤플레잉 기능에 "체험 모드" 파라미터 추가
- **AHA Moment 화면**: 신규 화면이나 기존 결제 전환 CTA 컴포넌트 재활용 가능
- **AI 톡 연계**: AHA Moment 완료 후 AI 톡 채팅방 자동 생성 + 첫 메시지 발송

## Phase 2: 고도화 로드맵

> Phase 1 MVP 완료 후 순차 착수. 각 기능은 Phase 1 완료 후 상세 스펙 작성 예정.

### 표현 코칭 (기존 기능 B)

**핵심 가치:** AI 톡 대화 중 사용자 입력에 대해 실시간으로 자연스러운 영어 표현을 제안하여, 대화 자체가 학습이 되는 경험 제공.

- 한글 입력 시 편하게/격식 2가지 영어 표현 인라인 제안
- 영어 입력 시 더 자연스러운 대체 표현 제안
- 제안 빈도 자동 조절 (핵심 표현에서만, 피로도 관리)

---

### Grow 리포트 (기존 P1-5)

**핵심 가치:** 통화 직후 CAFP 분석 리포트를 AI 톡으로 전달하여 "오늘 틀린 표현 → 내일 재연습" 루프를 완성하고, 월간 성장 리포트로 장기 동기 부여.

- 통화 종료 후 톡으로 CAFP 피드백 카드 전달
- 미니 복습 퀴즈 (빈칸 채우기)
- 월간 성장 리포트 (레이더 차트 + 성장 곡선)

---

### 장기 개인화 (기존 기능 C)

**핵심 가치:** 톡 대화 축적에 따라 Lv.1(범용) → Lv.4(코치 모드)까지 개인화 수준이 자동 상승하는 장기 학습 경로.

- Day 1~3: 기본 프로필 기반 범용 주제
- Week 1: 직무/관심사 맞춤
- Week 2~3: 약점 태그 기반 타겟 연습
- Month 1+: 성장 곡선 기반 코치 모드 전환

### Privacy 상세 설정 (기존 P2)

**핵심 가치:** AI 톡 대화 데이터의 투명한 제어로 사용자 신뢰 기반 구축.

- 동의 토글: AI만 보기(기본값) / 튜터도 보기 / 저장 안 함
- 보관 기간: 7일 / 30일(기본값) / 영구 + 즉시 삭제 버튼
- 민감정보 감지 시 "요약 저장 / 원문 삭제 / 저장 안 함" 선택
- "내 데이터 보기" 메뉴에서 AI 추출 컨텍스트 태그 확인/수정/삭제 가능
