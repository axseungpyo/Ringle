## 2. 핵심 문제 Top 3 선정

### Problem #1 (최우선): AI 전화영어 수신 거절률 40%
**문제 정의**
> 사용자가 스스로 예약한 AI 전화를 10번 중 4번 거절한다.
> 이는 학습 루틴 형성의 최대 병목으로, 결제 후 이탈의 직접 원인이다.

**근본 원인 (3-Why)**
- Why 1: 전화벨이 울려도 "지금은 아닌 것 같다"고 느낀다
- Why 2: 어떤 주제로 대화할지 전혀 모른 채 전화가 걸려와 심리적 준비가 안 돼 있다
- Why 3: 사용자의 일상/업무 맥락을 학습 재료로 전환하여 전화 전에 공급하는 구조가 없다

**연결 KPI**: 정상 연결률, 사용자 발신 비율, 리텐션, DAU

---

### Problem #2 (최우선): 무료 체험 중 AHA Moment 미발생 → 결제 전환 실패
**문제 정의**
> 7일 무료 체험 기간 동안 결제 전환의 핵심 드라이버인 롤플레잉을 경험하지 못하고 이탈한다.
> 롤플레이 경험자의 결제 전환율은 미경험자 대비 2.4배 높지만,
> 현재 온보딩 흐름은 이를 첫 경험으로 유도하지 않는다.

**근본 원인 (3-Why)**
- Why 1: 무료 체험 기간에 결제로 이어지지 않는다
- Why 2: 체험 기간에 "와, 이거 진짜 좋다"는 순간(AHA Moment)이 없다
- Why 3: 온보딩이 기능 소개 위주로 설계되어 있어, 가장 강력한 기능인 롤플레이에 도달하기 전에 이탈한다

**연결 KPI**: 결제 전환율 (최우선 KPI), 매출

---

### 구조적 공통 원인: 학습 경험 파편화 + 결정 피로

> Problem #1과 #2는 별개 문제가 아니라, 아래의 구조적 공통 원인에서 비롯된다.

**문제 정의**
> AI 튜터, AI 전화영어, 1:1 튜터 수업이 각각 독립적으로 운영되어
> 한 채널에서 학습한 내용이 다른 채널로 전이되지 않는다.
> 동시에 콘텐츠 과잉으로 "오늘 뭘 해야 하지?"라는 결정 피로가 학습 시작을 막는다.

**근본 원인 (3-Why)**
- Why 1: AI 튜터에서 연습한 표현이 AI 전화에서도, 튜터 수업에서도 재등장하지 않는다
- Why 2: 사용자의 실제 일상/업무 맥락이 학습 재료로 연결되지 않는다
- Why 3: 채널 간 데이터를 공유하는 컨텍스트 레이어가 존재하지 않는다

**연결 KPI**: AI Tutor → AI Phone 연속 활용률, 채널 간 학습 전이율, DAU/WAU, 스트릭(Streak) 유지율

## 3. 집중 서비스 결정

> **LCP(Learner Context Platform) 기반 AI 전화영어 + AI 튜터 롤플레잉 고도화**
> — 세 문제를 하나의 컨텍스트 레이어로 동시 해결

**결정 근거:**
- Problem #1(전화영어)과 Problem #2(롤플레잉)는 별개 문제가 아니라 **연결된 학습 루프** 문제다
- Problem #3(파편화/결정 피로)은 두 문제의 공통 원인이자 근본 구조 문제다
- **사용자의 일상 컨텍스트를 수집 → AI가 학습 재료로 변환 → 모든 채널에 동일하게 공급**하는
  LCP 구조가 세 문제를 동시에 해결한다
- RING LOOP는 LCP가 AI 전화영어에 적용되는 구체적 사용자 플로우이다

## 4. 솔루션 방향: LCP + RING LOOP

### 4.1 플랫폼 컨셉: LCP (Learner Context Platform)

> **"LCP — 일상이 곧 학습 재료가 되는 플랫폼"**
>
> 학습자가 자신의 일상·업무·감정을 능동적으로 입력하면
> AI가 이를 Topic Card, Roleplay Script, Phone Call Brief, Tutor Brief로 자동 변환하여
> AI 튜터 · AI 전화영어 · 1:1 튜터 수업 세 채널 모두에 동일한 컨텍스트로 공급한다.

### LCP 3계층 구조

```
[Context Capture] AI 톡 대화 레이어
  → AI 톡 (메인): AI가 톡으로 안부/근황을 물으며 자연스럽게 컨텍스트 수집
  → Daily Log (폴백): AI 톡 미사용 시 1분 구조화 입력 옵션
  → Event Pack: 미팅 아젠다, 이메일 초안, JD 등 업무 문서 붙여넣기

[Context Transformer] AI 변환 엔진
  → Topic Card: 오늘 토픽 3개 + 질문 트리 + 추천 표현 5개
  → Phone Call Brief: 통화 목적 1문장 + 예상 질문 5개 + 핵심 문장 2개
  → Roleplay Script: 상황·역할·목표·난이도 + 금지어/대체 표현
  → Tutor Brief: 배경 요약 + 목표 표현 + 민감 주제 제외 옵션

[Delivery] 채널별 실행 (동일 컨텍스트)
  → AI Tutor: Topic Card 기반 롤플레잉/디스커션
  → AI Phone: RING LOOP 플로우 (Ready → In → Grow)
  → 1:1 Tutor: Tutor Brief 전달 → 수업 후 표현 LCP 저장
```

### 4.2 AI 전화영어 구현 플로우: RING LOOP

> **"RING LOOP — 준비(Ready) → 통화(In) → 성장(Grow)"**
>
> LCP가 생성한 Phone Call Brief를 AI 전화영어에 적용하는 구체적 UX 루프.
> 심리적 장벽을 제거하고 발화 습관을 자동 형성한다.

```
[Ready] AI 톡 대화를 통한 자연스러운 전화 전환
  → AI 톡이 안부를 물으며 오늘의 학습 맥락 자연 도출
  → 대화 흐름에서 "이따 전화로 연습해볼까?" 자연스러운 전환 제안
  → "좋아! / 시간 변경 / 오늘은 패스" 선택
  → 전화 10분 전: 톡으로 Phone Call Brief + 핵심 문장 2개 브리핑

[In] AI 전화 수신
  → Phone Call Brief 기반 롤플레잉 캐릭터로 시작
  → 실시간 힌트 카드 (선택적, 자막 옵션 연동)
  → 발화량 게이지 (단어/문장 수 실시간 표시)

[Grow] 통화 종료 후
  → CAFP 분석 리포트 (5~10분 내)
  → 막힌 표현 1개 → LCP Memory 저장 → 내일 Roleplay Script에 자동 반영
  → 스트릭 연장 + 다음 전화 미리 예약 CTA
```

### 온보딩 개선 방향 (Problem #2 해결)

```
기존: 앱 설치 → 레벨테스트 → 기능 탐색 → (이탈)
개선: 앱 설치 → 레벨테스트 → 첫 롤플레잉 즉시 시작 (3분)
      → AHA Moment → "이 기능 더 쓰고 싶다" → 결제 유도
```

- **첫 화면 진입 후 3분 안에 롤플레잉 시작** — 선택이 아닌 기본 경로로 설계
- 7일 체험 목표: 롤플레잉 3회 이상 완료 → 전환율 극대화
- 체험 5일차: "롤플레이 3번 하셨어요! 전환율 2.4배 효과 문구 + 결제 CTA"

## 5. ux-designer 스펙 브리핑

### 설계 대상 기능 (우선순위 순)

| 단계 | 기능명 | 설명 |
|---|---|---|
| **MVP-01** | **AI 톡 (대화 + 컨텍스트 수집)** | AI가 톡으로 안부를 물으며 자연스럽게 일상/업무 컨텍스트 수집 (Daily Log는 폴백 옵션) |
| **MVP-02** | **AI 톡 → 전화 자연 전환 (Ready)** | 톡 대화 흐름에서 자연스럽게 전화 전환 제안 + Phone Call Brief 기반 브리핑 |
| **MVP-03** | **맥락형 AI 전화 롤플레이 (In)** | 3개 기본 캐릭터(면접관/동료/클라이언트)와 롤플레이 대화 |
| **MVP-04** | **스마트 거절 UX** | "5분 뒤 / 오늘은 건너뛰기 / 내가 먼저 전화" 옵션 (거절 → 재참여 전환) |
| **MVP-05** | **롤플레잉 온보딩** | 레벨테스트 직후 3분 롤플레잉 체험 → AHA Moment → 결제 CTA |
| Phase 2 | Grow 리포트 + 표현 코칭 | CAFP 분석 + 실시간 표현 제안 + 월간 성장 리포트 |
| Phase 2 | 장기 개인화 + Privacy | Lv.1~4 개인화 + 데이터 동의/보관 설정 |

### 설계 필수 조건

- 각 기능 **Before/After 비교 스펙** 포함
- 주요 화면 **최소 5개** 와이어프레임 텍스트 기술
- 기대 KPI: 수신 거절률 40% → 25%(보수적) / 20%(낙관적), 롤플레잉 온보딩 경험률 → 80% 이상
- 기술 구현 범위: 현 상용 LLM + 음성인식 기술 수준 (추가 R&D 없이 구현 가능한 범위)

## 6. doc-writer 스펙 브리핑

### PPT 스토리라인 방향

```
1부. 시장과 기회 (시장 분석 기반)
  → AI 영어 시장 급성장 + 링글의 유일한 포지션

2부. 지금 링글의 문제 (서비스 분석 기반)
  → 수신 거절 40% + AHA Moment 미발생 = 성장의 병목

3부. 솔루션: RING LOOP
  → 준비(Ready) → 통화(In) → 성장(Grow) 루프
  → Before/After 화면 비교

4부. 기대 효과 및 실행 계획
  → 수신 거절률 20%↓ / 결제 전환율 X배 / 시장 점유율 목표

```

### 컨셉보드 핵심 메시지 방향
- 슬로건 후보: **"링글이 먼저 전화합니다, 준비됐을 때"**
- 비주얼 방향: 직장인이 스마트폰으로 AI와 롤플레잉하는 장면 + RING LOOP 그래픽
- 감성 키워드: 자신감, 실전, 비즈니스 성장, 습관
