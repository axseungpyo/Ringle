## 2. 핵심 문제 정의

### 핵심 문제 (한 문장)

> **"사용자가 스스로 예약한 AI 전화를 40% 거절하는 이유는, 전화 전에 무엇을 말할지 준비되지 않았기 때문이다."**

이것은 단순한 UX 불편이 아니라 **매출·리텐션·DAU를 직접 좌우하는 최대 병목**이다.
거절이 반복되면 학습 루틴이 형성되지 않고, 결제 후에도 방치·이탈로 이어진다.

### 근본 원인 분석 (3-Why)

```
Why 1: 전화벨이 울려도 "지금은 아닌 것 같다"고 느낀다
  ↓
Why 2: 어떤 주제로 대화할지 전혀 모른 채 전화가 걸려온다 (맥락 부재 + 갑작스러움)
  ↓
Why 3: 사용자의 일상/업무 맥락을 전화 전에 수집하여 학습 재료로 전환하는 구조가 없다
```

### 연결 KPI (최우선 순)

| KPI | 현재 | 목표 | 이유 |
|---|---|---|---|
| AI 전화 수신 거절률 | **40%** | **25% 이하** | 학습 루틴 형성의 직접 지표 |
| 사용자 발신 비율 | 22.3% | 30% 이상 | 능동적 학습 전환의 지표 |
| 결제 전환율 | 미확인 | 롤플레이 경험자 2.4배 | 매출 직결 (최우선 KPI) |
| 리텐션 / DAU | 기준 미확인 | 개선 | 장기 고객 가치 |

### 부가 문제: 온보딩에서 AHA Moment 미발생

> 핵심 문제(거절률)를 해결하는 과정에서, 신규 유저의 첫 경험도 함께 설계한다.

- 롤플레이 경험자의 결제 전환율이 미경험자 대비 **2.4배** 높음
- 그러나 현재 온보딩은 기능 소개 위주 → 롤플레이 도달 전 이탈
- **솔루션의 핵심 기능(AI 톡 + 롤플레이)을 온보딩 첫 경험으로 배치**하면 자연스럽게 해결

## 3. 집중 서비스 결정

> **집중 대상: AI 전화영어**
> **솔루션 수단: AI 톡 (페르소나 기반 관계형 인터페이스)**

**결정 근거:**
- AI 전화영어 수신 거절률 40%는 **매출·리텐션·DAU를 동시에 잠식하는 단일 최대 병목**이다
- 근본 원인은 "맥락 부재 + 심리적 준비 부족" → **전화 전에 관계와 맥락을 만들어주면 해결**된다
- AI 톡이 친구처럼 먼저 대화하여 일상/업무 맥락을 자연스럽게 수집하고, 이를 전화 시나리오로 전환한다
- AI 톡은 독립 서비스 개선이 아니라 **AI 전화영어의 거절률을 해결하기 위한 솔루션 수단**이다
- 부가 효과로, AI 톡에서의 롤플레이 경험이 **온보딩 AHA Moment**까지 자연스럽게 연결된다

## 4. 솔루션 방향: RING LOOP

### 솔루션 한 문장

> **"AI 톡이 친구처럼 먼저 대화하여 맥락을 만들고, 자연스럽게 전화로 연결한다."**

### RING LOOP — 준비(Ready) → 통화(In) → 성장(Grow)

AI 전화영어의 수신 거절률 40% 문제를 해결하는 **단일 UX 루프**.
핵심은 "갑자기 걸려오는 전화"를 "준비된 대화의 연장"으로 바꾸는 것이다.

```
[Ready] AI 톡이 먼저 대화한다
  → 페르소나를 가진 AI가 톡으로 안부/근황을 물으며 관계를 형성
  → 대화 속에서 사용자의 일상/업무 맥락을 자연스럽게 수집
  → "이따 전화로 연습해볼까?" 자연스러운 전환 제안
  → 전화 10분 전: 톡으로 오늘의 주제 + 핵심 문장 브리핑
  ※ 사용자는 전화가 오기 전에 이미 무엇을 말할지 알고 있다

[In] 맥락이 이어지는 전화
  → 톡에서 나눈 이야기가 그대로 전화 시나리오로 연결
  → AI가 면접관/동료/클라이언트 캐릭터로 롤플레이 진행
  → 실시간 힌트 카드 (선택적)
  ※ "AI와 통화하는 어색함" → "친구와 이어서 대화하는 자연스러움"

[Grow] 다음 대화를 준비한다
  → 통화 직후 CAFP 분석 리포트
  → 막힌 표현 → 내일 톡 대화와 전화 시나리오에 자동 반영
  → 스트릭 연장 + 다음 전화 예약 CTA
  ※ 오늘의 약점이 내일의 학습 재료가 된다
```

### AI 톡의 역할 (솔루션 수단으로서)

> AI 톡은 독립적인 서비스 개선 대상이 아니라, **AI 전화영어의 거절률을 낮추기 위한 솔루션 수단**이다.

> ⚠️ **핵심 설계 원칙**: 링글의 기존 서비스(레벨테스트, AI 튜터, AI 전화영어) 위에서 작동한다.
> 기존 기능을 다시 만들지 않으며, RING LOOP는 기존 서비스에 **얹히는 새로운 레이어**이다.

```
┌─────────────────────────────────────────────────┐
│              AI 톡 = 전화 준비 인터페이스         │
│                                                   │
│  역할 1. 친구    페르소나 기반 관계 형성 + 안부    │
│  역할 2. 준비    대화 속에서 전화 맥락 자연 도출   │
│  역할 3. 데이터  개인 맥락 수집 → 전화 시나리오화  │
│                                                   │
│  ★ 데이터 수집 방식: 설문/폼이 아닌,              │
│    대화 흐름 안에서 자연스럽게 수집               │
│    → 사용자가 AI 톡을 쓰고 싶은 명분이 핵심      │
│                                                   │
│  결과: "갑자기 오는 전화" → "준비된 대화의 연장"   │
│        → 수신 거절률 40% → 25% 이하               │
└─────────────────────────────────────────────────┘
```

### AI 톡 대화형 맥락 수집 설계

> AI 톡은 대화를 통해 사용자의 학습 맥락을 자연스럽게 파악하고, 이 맥락이 전화 시나리오로 전환된다.
> 대화할수록 전화가 더 **"나를 위한 전화"**가 되므로, 사용자는 AI 톡을 쓸 명분이 생기고, 전화를 받을 준비가 된다.

**원칙**: 설문/폼 X → 일상 대화 속에서 자연스럽게 파악

| 수집 데이터 | 대화 맥락 예시 | 전화 시나리오 활용 |
|---|---|---|
| **직무·업무 환경** | "요즘 어떤 일 해요?" "글로벌 팀이랑 일해요" | 시나리오 주제 선정 (팀 회의, 이메일, 발표 등) |
| **협업 대상국·문화권** | "어느 나라 사람들이랑 일해요?" | 억양·문화 맞춤, 상황별 뉘앙스 반영 |
| **핵심 영어 사용 상황** | "이메일? 회의? 발표? 뭐가 제일 많아요?" | 훈련 영역 특정 (회의 중심 vs 이메일 중심) |
| **구체적 약점** | "어떤 부분이 제일 불안해요?" | 맞춤 표현 추천, 힌트 카드 난이도 조절 |
| **학습 목표·동기** | "영어 잘하면 뭐가 제일 하고 싶어요?" | 목표 기반 커리큘럼 (출장/이직/승진) |
| **데드라인** | "다음 달에 미국 출장이 있어서..." | 학습 일정·강도 설계 |
| **선호 시간대** | "보통 언제가 편해요? 아침? 밤?" | AI 전화 발신 시간 최적화 |
| **과거 학습 이력** | "영어 공부 해봤어요? 어떤 방식?" | 이탈 방지 전략 (기존 실패 원인 회피) |
| **성격·학습 스타일** | "소리내서 말하진 못했지만요" | 난이도·피드백 톤 조절 |
| **감정 상태** | "오늘 회의 좀 긴장돼요 😅" | 당일 시나리오 난이도·격려 수준 조절 |

**핵심 선순환 구조:**
```
AI 톡 대화 → 학습 맥락 축적 → 맞춤 전화 시나리오 생성
     ↑                                    ↓
  피드백 반영 ← 통화 결과 분석 ← 맞춤 전화 수신/발신
```

- 대화할수록 전화가 정확해지므로 **AI 톡 참여율 상승** (사용 명분)
- 전화가 맞춤형이므로 **수신 거절률 하락** (준비된 느낌)
- 통화 결과가 다음 AI 톡에 반영되므로 **리텐션 강화** (성장 실감)

### 부가 효과: 온보딩 AHA Moment

RING LOOP의 핵심 기능(AI 톡 + 롤플레이)을 신규 유저 온보딩에도 적용하여,
**첫 3분 안에 AHA Moment를 발생**시키고 결제 전환율을 높인다.

```
기존: 앱 설치 → 레벨테스트 → 기능 탐색 → (이탈)
개선: 앱 설치 → 레벨테스트 → 첫 롤플레잉 즉시 시작 (3분)
      → AHA Moment → AI 톡 첫 대화 → 결제 유도
```

- 롤플레이 경험자 결제 전환율 **2.4배** → 첫 경험으로 반드시 유도
- 7일 체험 목표: 롤플레잉 3회 이상 완료
- 체험 후 AI 톡 대화 시작 → RING LOOP 습관 형성

## 5. ux-designer 스펙 브리핑

### 설계 대상 기능 (우선순위 순)

> 모든 기능은 **"AI 전화영어 수신 거절률 40% → 25%"**라는 단일 목표를 향한다.

| 단계 | 기능명 | RING LOOP 위치 | 설명 |
|---|---|---|---|
| **MVP-01** | **AI 톡 (페르소나 대화)** | Ready 핵심 | AI가 친구처럼 톡으로 대화하며 전화 맥락을 자연스럽게 준비 |
| **MVP-02** | **톡 → 전화 자연 전환** | Ready → In 연결 | 톡 대화 흐름에서 자연스럽게 전화 제안 + 직전 브리핑 |
| **MVP-03** | **맥락형 AI 전화 롤플레이** | In | 톡에서 나눈 맥락이 그대로 전화 시나리오로 연결 |
| **MVP-04** | **스마트 거절 UX** | In (이탈 방지) | 거절 → 재스케줄링으로 전환하여 학습 연속성 유지 |
| **MVP-05** | **롤플레잉 온보딩** | 부가 효과 | 첫 3분 롤플레이 → AHA Moment → AI 톡 습관 형성 |
| Phase 2 | Grow 리포트 + 표현 코칭 | Grow | CAFP 분석 + 약점 → 내일 시나리오 반영 |
| Phase 2 | 장기 개인화 + Privacy | 전체 | Lv.1~4 개인화 + 데이터 동의/보관 설정 |

### 설계 필수 조건

- 각 기능 **Before/After 비교 스펙** 포함
- 주요 화면 **최소 5개** 와이어프레임 텍스트 기술
- 기대 KPI: 수신 거절률 40% → 25%(보수적) / 20%(낙관적), 롤플레잉 온보딩 경험률 → 80% 이상
- 기술 구현 범위: 현 상용 LLM + 음성인식 기술 수준 (추가 R&D 없이 구현 가능한 범위)

## 6. doc-writer 스펙 브리핑

### PPT 스토리라인 방향

```
1부. 문제 발견 — AI 전화영어, 40%가 거절한다
  → 데이터: 수신 거절률 40%, 근본 원인 3-Why 분석
  → "왜 스스로 예약해놓고 거절하는가?"

2부. 원인 진단 — 준비 없이 걸려오는 전화
  → 맥락 부재 + 심리적 준비 부족 + 갑작스러움
  → 경쟁사 비교: 스픽은 발화량, 링글은 하이브리드가 강점인데 전화가 병목

3부. 솔루션 — RING LOOP: 대화하고, 준비되면, 전화한다
  → AI 톡이 친구처럼 먼저 대화 → 맥락 도출 → 자연스러운 전화 전환
  → Ready → In → Grow 루프 + Before/After 화면 비교

4부. 기대 효과 — 6개월 내 측정 가능한 KPI
  → 거절률 40%→25% / 사용자 발신 22%→30% / 결제 전환율 2.4배
  → 부가 효과: 온보딩 AHA Moment + 롤플레이 경험률 80%
```

### 컨셉보드 핵심 메시지 방향
- 슬로건 후보: **"대화하고, 준비되면, 전화한다 — RING LOOP"**
- 한눈에 보이는 구조: 문제(거절률 40%) → 솔루션(AI 톡→전화) → 효과(25%↓)
- 비주얼: 직장인의 AI 톡 대화 → 자연스러운 전화 전환 흐름
- 감성 키워드: 자신감, 준비된 대화, 습관 형성, 실전
