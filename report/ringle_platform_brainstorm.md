# Ringle AI 스피킹 고도화 브레인스토밍 (플랫폼화: Learner Context Platform)

작성자: 홍승표(승표님)  
목표: **학습자(고객) 주도 데이터 입력 → 개인화 학습 재료 생성 → AI 튜터/AI 전화/튜터 수업 전 채널 통합**을 통해 Ringle의 AI 스피킹 경험을 “플랫폼”으로 확장

---

## 1. 문제 인식 (Why now)

### 1) AI 전화영어 거절(수신 거부) 리스크

- “전화를 받을 이유”가 즉시 납득되지 않으면 **침범감/불편**으로 거절 가능성이 커짐
- 사용자가 *원하는 시간·상황·주제*가 맞지 않으면, 기능 자체의 효용과 무관하게 이탈 발생

### 2) 학습 경험의 파편화

- AI로 말한 내용이 1:1 튜터 수업으로 이어지지 않거나  
  1:1 수업에서 다룬 내용이 AI 연습으로 전이되지 않으면 **학습 효율이 떨어지고 몰입이 끊김**

### 3) 토픽 선택 비용(결정 피로)

- 콘텐츠가 많을수록 선택 부담이 커져서  
  “오늘은 뭘 해야 하지?” → **학습 시작 지연** → **습관 형성 실패**로 이어질 수 있음

---

## 2. 핵심 아이디어 (Big Idea)

## Learner Context Platform (LCP)

학습자가 **자신의 일상/업무/감정/이벤트**를 능동적으로 입력하고,  
이를 기반으로 AI가 **토픽/질문 트리/롤플레이/전화 브리프/튜터 브리프**를 자동 생성하여  
AI 튜터 · AI 전화 · 1:1 튜터 수업에 **동일한 컨텍스트**로 공급하는 “개인화 학습 플랫폼”.

> 일기 자체가 목적이 아니라,  
> **일상 데이터 → Speaking Material로 변환하는 파이프라인**이 핵심.

---

## 3. 플랫폼 구조 (Modules)

### A. Context Capture (사용자 입력 레이어)

**1~2분 입력**으로 “오늘의 대화 재료”를 확보

- **Daily Log (기본)**
  - 오늘 있었던 일(자유서술)
  - 감정(선택: 😊😐😣 등)
  - 내일 일정(선택)
  - 말해보고 싶은 주제(선택)
  - 난이도(쉬움/보통/어려움)

- **Event Pack (업무 특화, 선택)**
  - 미팅 아젠다, 이메일 초안, JD, PT 초안 등 “문서 붙여넣기”

- **Voice Note (선택)**
  - 30초 음성 메모 → 텍스트 변환 후 가공

> 플랫폼화를 위해 **구조화 필드 + 자유서술 1칸** 조합을 권장  
> (자유서술만 있으면 가공/측정/반복학습이 어려움)

---

### B. Context Transformer (AI 변환 엔진)

입력된 컨텍스트를 다음 “산출물 카드(카드형 객체)”로 자동 생성

- **Topic Card**
  - 오늘 토픽 3개(우선순위 포함)
  - 각 토픽별 질문 트리(Depth 3)
  - 추천 표현/어휘 5개

- **Roleplay Script**
  - 상황/역할/목표/난이도
  - 금지어(직역 방지) & 대체 표현

- **Phone Call Brief**
  - 통화 목적 1문장
  - 예상 질문 5개
  - 오늘의 “핵심 문장 2개”(프리콜 워밍업용)

- **Tutor Brief (Zoom/1:1용)**
  - 배경 요약(튜터가 10초 안에 파악)
  - 오늘 목표(발화 목표/교정 포인트)
  - 민감 주제 제외 옵션 반영

---

### C. Delivery (채널별 실행 레이어)

동일 컨텍스트가 **AI Tutor → AI Phone → Tutor 수업**을 관통

- **AI Tutor (롤플레잉/디스커션)**
  - Topic Card/Roleplay Script를 “오늘의 추천”으로 1클릭 시작

- **AI Phone English (인바운드/아웃바운드 콜)**
  - 수신 화면/푸시에서 “오늘 통화 목적” 1문장 노출
  - 사용자가 즉시 선택:
    - [지금 받기] [10분 뒤] [오늘은 텍스트/음성메모로 대체] [주제 바꾸기]
  - **프리콜 워밍업(1분)**: 핵심 문장 2개만 미리 확인

- **Tutor Zoom/1:1**
  - 수업 시작 전 Tutor Brief 전달
  - 수업 후: 실제로 막혔던 문장/교정 표현을 LCP에 저장 → 다음 AI 연습으로 자동 생성

---

### D. Memory & Consent (신뢰/동의 레이어)

개인 데이터 기반 서비스의 핵심은 “신뢰” (투명한 통제 제공)

- **공유 토글**
  - AI만 보기 / 튜터도 보기 / 저장 안 함(세션 후 자동 삭제)
- **민감정보 안내**
  - 건강/개인정보/강한 감정 등 감지 시 요약 저장/삭제 옵션 제공
- **보관 기간**
  - 7일/30일/영구 선택 + “즉시 삭제” 버튼

## 4. 대표 사용자 플로우 (User Flows)

### Flow 1) Daily Log → AI Tutor 대화

1. 홈: “오늘의 1분 로그”
2. 입력 후 “오늘의 카드 생성”
3. AI Tutor에서 Topic Card 선택 → 즉시 대화 시작
4. 대화 종료 시 “막힌 문장 1개 저장” → 내일 복습 카드 자동 생성

### Flow 2) Daily Log → AI Phone (거절률 감소 플로우)

1. 사용자가 “오늘 전화 연습 허용” 토글 ON + 시간대 설정
2. 전화 푸시/수신 화면에 “오늘의 목적 1문장” 노출
3. [지금/10분 뒤/대체] 선택
4. 통화 후 “오늘의 개선 포인트 2개” 저장 → 다음날 워밍업으로 연결

### Flow 3) Event Pack → Tutor Zoom 수업

1. 사용자가 회의 아젠다/발표 초안 업로드
2. Tutor Brief 생성(핵심 요약 + 목표 표현)
3. 튜터 수업에서 해당 문서 기반 roleplay 진행
4. 수업 종료 후 “실전 표현 팩” 자동 생성 → AI Tutor로 반복 연습

---

## 5. MVP 범위 (6~8주 기준)

### MUST (필수)

- Daily Log (구조화 필드 + 자유서술)
- Topic Card + Phone Call Brief 생성
- AI Phone: [지금/10분 뒤/대체] 선택 UX
- Tutor Brief 최소 버전(요약/목표/금지어)

### SHOULD (권장)

- 프리콜 워밍업(핵심 문장 2개)
- 막힌 문장 1개 저장 → 내일 roleplay 자동 생성

### COULD (후속)

- Voice Note 입력
- 직군/목표(면접/업무/시험) 템플릿 팩
- 조직(B2B)용 템플릿 배포

## 6. KPI / 실험 설계

### 핵심 KPI

- **AI 전화 수신 거절률 감소**
  - (가설) “통화 목적이 개인 컨텍스트와 연결되면 거절률이 감소”
- **통화 완료율(중도 이탈률) 개선**
- **DAU/WAU 및 연속 학습일(Streak) 증가**
- **AI Tutor → Tutor 수업 전이율**
  - (예) AI 대화 후 7일 내 1:1 예약률
- **재구독/연장율**

### A/B 테스트 아이디어

- A: 기존 인바운드 콜 UX  
  vs  
  B: “오늘 목적 1문장 + [지금/10분 뒤/대체]” UX
- B군에서 통화 길이/완주율/거절률 변화 측정

---

## 7. 리스크 & 대응

- **프라이버시 우려**
  - 강한 동의/삭제/보관기간 옵션 + 민감정보 요약 저장
- **사용자 입력 귀찮음**
  - 1분 입력, 기본값, 음성 입력, “한 줄만 써도 카드 생성” 설계
- **토픽 품질 편차**
  - 사용자 선택형(추천 3개) + “주제 리롤/난이도 조절” 제공
- **튜터 활용 저조**
  - Tutor Brief를 “수업 시작 체크리스트” 형태로 제공해 습관화
